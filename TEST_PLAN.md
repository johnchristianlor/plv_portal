# Test Plan

- Student cannot access admin assessment APIs.
- Admin cannot be impersonated with localStorage.
- One student cannot read another student's attempt.
- Correct answers are not present in active attempt responses.
- Expired or modified JWTs are rejected.
- SQL injection strings are treated as data.
- XSS payloads render as text in assessment titles/questions.
- Timer survives refresh and uses server deadline.
- Duplicate start requests resume the existing active attempt.
- Autosave recovers after temporary network loss.
- Tab hidden, blur, paste, print, and fullscreen exit create incidents.
- Incident thresholds are reviewed as signals, not automatic proof.
- Published assessment versions remain immutable.
- Review shows the exact version answered after release.
- Essay answers can be manually graded.
- Final scores sync to Supabase summaries.
- No plaintext passwords remain after migration.
- No service-role, Turso, Backblaze, or Cloudflare secret exists in frontend files.
- Private Backblaze files require authenticated Functions.
- Direct admin URL navigation by a student returns 403 from APIs.
- Mobile assessment layout works.
- Existing grades, attendance, activities, and file sharing still load.
